# 模組指南：使用者介面層 (Adapter Layer)

本模組為系統的流量入口與通訊邊界，主要負責外部協定的適配、端點暴露以及輸入參數的解析與基本校驗。本層級不處理任何實質的業務邏輯，僅作為外部世界與系統內部的轉譯橋樑。

## 核心職責

* 外部協定適配：將外部進入的各種通訊協定請求轉化為系統內部的調用指令。
* 參數解析與驗證：負責前端或外部傳入參數的初步校驗，確保進入內部架構的資料符合基本格式要求。
* 回應格式封裝：將內部處理完畢的結果或拋出的異常，轉譯為標準的對外響應格式與適當的網路狀態碼。

## 包含組件

* 網路控制器 (Controllers)：處理 HTTP 與 RESTful 請求的轉發與端點暴露，落實資源命名與版本控制。
* 訊息監聽器 (MQ Consumers)：監聽並接收外部訊息佇列的事件訊息，進行初步解析後交由內部處理。
* 排程觸發器 (Job Triggers)：承接定時任務或外部調度系統的觸發入口。

## 相依性約束與方向

* 允許依賴：僅可依賴應用層 (Application Layer) 以及公共合約與客戶端模組 (Common/Client DTO)。
* 嚴格禁止：本模組絕對禁止繞過應用層直接操作領域層或基礎設施層。

## 設計防線與禁忌

* 業務邏輯零容忍：本層級內嚴禁包含任何業務規則、狀態轉變或交易宣告。所有的業務編排與程序必須交由應用層專屬的 Executor 處理。
* 資訊安全與防腐：控制器不應直接將深層的核心模型或敏感的資料庫實體暴露給外部客戶端，必須透過 DTO 建立清晰的資料交換邊界，以維護系統的內在安全性。
