# 模組指南：基礎設施層 (Infrastructure Layer)

本模組為系統的技術實現與外部資源的插件化支援中心，負責將領域層定義的抽象介面進行具體技術實作，並處理所有與外部技術框架、資料庫或第三方服務的溝通細節。

## 核心職責

* 技術實現的插件化支援：封裝所有具體的底層技術細節，使核心業務邏輯不受外部框架更迭的影響。
* 資料庫存取實作：實現領域層定義的儲存庫介面，處理資料的持久化與檢索。
* 對外呼叫實作：實現遠端程序呼叫介面，處理與外部第三方服務的通訊。
* 設定載入：管理與載入系統所需的各類技術組件配置。

## 包含組件

* 儲存庫實現類別 (Repository Impl)：實作領域儲存庫介面，負責在資料庫與領域物件之間進行轉譯。
* 持久化實體 (JPA Entity / Data Object)：專門用於與資料庫資料表結構進行映射的資料物件。
* 閘道實現類別 (Gateway Impl)：負責外部 RPC 或 HTTP 客戶端的具體對外呼叫實作。
* 資料查詢組件：在讀寫分離架構下，可直接使用原生 SQL、JPA Projections 或 MyBatis 繞過領域層提供高效查詢。

## 相依性約束與方向

* 允許依賴：本模組依賴領域層 (Domain Layer) 以及各類第三方技術框架。
* 被依賴性：本模組的抽象介面由應用層調度，具體技術實作則以插件形式支撐整個系統。

## 設計防線與最佳實踐

* 三層映射模式：核心業務領域中僅存在純 Java 物件，基礎設施層則定義專屬的持久化實體，並由儲存庫實現類別負責重建與解構操作，徹底隔離資料庫結構與業務邏輯。
* 避免技術滲透：嚴格禁止將本層的技術細節（如 JPA 標註、ORM 框架特性或 SQL 語法）滲透、暴露或直接使用於領域層中。
* 讀寫分離優化：針對純粹的查詢需求，允許繞過領域模型，直接在本層將資料庫檢索結果映射為專門的唯讀模型或 DTO 供表現層呈現。
